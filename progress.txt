## Search Persistence Implementation Progress

### Completed Tasks

1. **Prisma Schema & Client Setup** (Task 1-2)
   - Created full Prisma schema with all models: User, Search, Video, VideoMention, VideoHashtag, VideoStats, SearchResult, BrandExplorerSummary, BrandExplorerInfluencer
   - Fixed prisma.config.ts paths for Prisma 7 compatibility
   - Created Prisma client singleton at server/lib/prisma/ with PrismaPg adapter
   - Pushed schema to Supabase database

2. **Keyword Search History Service** (Task 3)
   - Created server/services/keyword-search-history/ with types, implementation, and index
   - Implemented: findKeywordSearch, saveKeywordSearch, refreshKeywordSearch, getKeywordSearchResults
   - Uses connectOrCreate for placeholder user
   - Handles video upsert and stats snapshot creation

3. **Brand Explorer History Service** (Task 4)
   - Created server/services/brand-explorer-history/ with types, implementation, and index
   - Implemented: findBrandExplorerSearch, saveBrandExplorerSearch, refreshBrandExplorerSearch, getBrandExplorerResults
   - Handles summary, influencers, and video relationships
   - Creates stats snapshots for historical tracking

4. **Modify POST /api/search** (Task 5)
   - DB-first approach: checks for existing search before calling Apify
   - Returns cached results with `cached: true` flag if found
   - Saves new searches to database after Apify + classification
   - Returns `searchId` in response for future reference

5. **Modify POST /api/brand-explorer** (Task 6)
   - DB-first approach: checks for existing search before calling Apify
   - Returns cached results with `cached: true` flag if found
   - Saves new searches to database after exploration
   - Returns `searchId` in response for future reference
   - Fixed route path (was `/api/api/brand-explorer`, now `/api/brand-explorer`)

6. **Create GET /api/history endpoint** (Task 7)
   - Returns list of all searches for current user
   - Includes: id, type, query, createdAt, updatedAt, resultCount, summary
   - Ordered by updatedAt descending (most recent first)

7. **Create GET /api/history/:id endpoint** (Task 8)
   - Returns full results for a specific search
   - Handles both keyword and brand explorer types
   - Delegates to service functions for proper response format

8. **Create POST /api/history/:id/refresh endpoint** (Task 9)
   - Looks up search by ID to determine type (keyword or brand_explorer)
   - Re-runs Apify search + classification for keyword searches
   - Re-runs brand exploration for brand explorer searches
   - Uses refreshKeywordSearch/refreshBrandExplorerSearch to update DB
   - Returns fresh results in same format as original endpoints
   - Proper error handling for Apify, BrandExplorer, and internal errors

9. **Frontend Integration** (Task 10)
   - Created useHistoryQuery hook to fetch search history from GET /api/history
   - Created useHistoryDetailQuery hook to load results by search ID
   - Created useRefreshMutation hook for POST /api/history/:id/refresh
   - Updated Sidebar component to use API-backed history with loading state
   - Updated index.tsx to use searchId for loading cached results
   - Added searchId and cached fields to SearchResponse and BrandExplorerResponse types
   - Replaced localStorage-based history with persistent database storage

### Implementation Complete

All tasks from the PRD have been implemented:
- Backend: Database schema, services, and API endpoints
- Frontend: Hooks and components wired to use the new API

The old useSearchHistory hook (localStorage) can be removed if no longer needed.
